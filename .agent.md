# ViOS: Project Memory

## Project Goal
Build a "Hybrid" Operating System (**ViOS**) based on the Edge-to-Cloud model.
- **Robot (Edge)**: `no_std` Microkernel, Rust Cells, Real-time focus.
- **Server (Cloud)**: Full Hybrid OS, Cloud-Hypervisor, AI Hub.
- **The Glue**: Distributed IPC (dIPC) making the network transparent.

## Architecture Directives
1.  **Core**: Microkernel (Rust/seL4) - Safety.
2.  **Layer 1.5 (FrameKernel)**: Strict separation of Safe/Unsafe (Asterinas model).
3.  **Layer 2 (Cells)**: Dynamic Relocatable Object Files aka "Cells" (Theseus model) - Real-time Performance & Live Updates.
4.  **Layer 2.5 (Drivers)**: Silos/VMM with Zero-Copy "Grant" IPC (Tock model).
5.  **Layer 3 (Apps)**: Universal Compatibility (POSIX/WSL1/WSL2).
6.  **Connectivity**: Distributed IPC.
7.  **Build**: Monorepo "Shapeshifter".
8.  **Graphics**: Userspace Compositor + Wayland Bridge.

## Development Standards
### 1. Intent & Documentation
- Every `pub` item must have a docstring (`///`) explaining **why** it exists, not just what it does.
- Link to architecture docs (`docs/architecture/`) where applicable.

### 2. Safety (FrameKernel Rule)
- `unsafe` is **FORBIDDEN** outside `kernel/src/arch` and `hal/`.
- Every `unsafe` block must have a `// SAFETY:` comment.

### 3. Error Handling
- No panics (`unwrap`/`expect`) in critical paths.
- Use `Result<T, SystemError>` and propagate errors.

### 4. Code Quality
- Must be `cargo clippy` clean.
- Must run `cargo fmt`.
- Mock hardware for tests (`MockHardware` trait).

## Key Technologies & Codebase Context
- **Language**: Rust `no_std` (Kernel), `std` (Cloud/Tools).
- **Core Crates**:
    - `kernel`: The microkernel core.
    - `ostd`: User/Driver-space safe framework (like `std` but for ViOS).
    - `hal`: Hardware Abstraction Layers (`hal-core`, `hal-riscv`).
- **IPC Mechanism**: Synchronous Rendezvous (Hubris-style) + Zero-Copy Grants (Tock-style).
    - **Syscalls**: `Send`, `Recv`, `Reply`, `Yield`, `Spawn`, `Grant`, `Map`.
    - **Leases**: `sys_lend`, `sys_borrow` for safe memory sharing.
- **Concurrency**:
    - **Tasks**: Preemptive, Priority-based.
    - **Threads**: Dynamic spawning supported (`ostd::task::spawn`).
    - **Async**: `Embassy` based executor.
