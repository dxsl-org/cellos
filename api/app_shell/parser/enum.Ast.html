Top-level abstract syntax tree for one shell line.

Empty input.

A single simple command.

§

Background(Cmd)

cmd & — run in background.

§

Sequence(Vec<Ast>)

cmd1 ; cmd2 — sequential execution.

§

And(Box<Ast>, Box<Ast>)

cmd1 && cmd2 — run cmd2 only if cmd1 exits 0 (success).

§

Or(Box<Ast>, Box<Ast>)

cmd1 || cmd2 — run cmd2 only if cmd1 exits non-zero (failure).

While

while COND; do BODY; done — loop while COND exits 0.

FuncDef

name() { body; } — define a shell function.

The body is stored as a reconstructed string; the executor registers it in the function table so later invocations of name run the body.

Fields

§name: String
§body: String
§

Case

case EXPR in pattern) BODY ;; … esac — pattern-match dispatch.

expr is the unexpanded word after case; expansion happens at execute time. Patterns are exact strings or * (wildcard). Note: the whole statement must be on a single line or in a single source-script line; mixing with ;-sequences on the same line is not supported (the outer ; split would fragment the ;; markers).

Fields

§expr: String
§arms: Vec<(String, Box<Ast>)>
§

For

for VAR in word1 word2 …; do BODY; done — iterate over a word list.

Sets $VAR to each word in order, runs BODY, then advances. $VAR expansion in BODY uses the same static var store as VAR=value.

Fields

§var: String
§words: Vec<String>
§body: Box<Ast>
§

If

if COND; then BODY; fi — conditional execution.

cond exit-code 0 → run then_b; non-zero → run else_b if present.
