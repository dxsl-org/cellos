Module pmp 

Expand description

RISC-V PMP (Physical Memory Protection) region definitions.

§Architecture Constraint

PMP CSRs (pmpaddr*, pmpcfg*) are M-mode-only. Writing them from S-mode raises an Illegal Instruction trap. ViCell runs in S-mode under OpenSBI, so this module cannot write PMP registers at runtime.

This file documents the intended PMP layout for use by a future custom M-mode firmware shim (Phase 32+). The KERNEL_PMP_REGIONS slice encodes the desired configuration that the shim should apply before mret into S-mode.

§QEMU virt PMP budget

  • 16 total entries; OpenSBI claims 2 (entry 0 = firmware deny, entry 1 = all-access)
  • 14 free entries for ViCell (entries 2–15)
  • Entries 2–6 used for kernel + MMIO protection (see KERNEL_PMP_REGIONS)
  • Entries 7–15 reserved for per-Cell isolation (Phase 32)

§NAPOT encoding

pmpaddr = (base >> 2) | (size/8 - 1) Requirements: base aligned to size; size must be a power of two ≥ 8.
