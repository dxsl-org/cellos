Expand description

HKDF-SHA256 (RFC 5869) — extract + expand, hand-rolled over the local super::sha256 implementation.

Not the hkdf crate on purpose (dossier-4 Decision 1 / YAGNI+KISS): SHA-256 is already local, HKDF-SHA256 is ~40 lines on top of it, and this must build in kernel-adjacent no_std contexts where a new dependency is unwanted.

Functions§

expand
HKDF-Expand: fills okm (any length up to 255 * HASH_LEN, RFC 5869 §2.3) with T(1) || T(2) || ..., T(i) = HMAC-Hash(PRK, T(i-1) || info || i). info must fit in MAX_INFO_LEN bytes (32) — generous for this crate’s only use (empty info in cdi::derive_cdi); asserts rather than truncates.
extract
HKDF-Extract: PRK = HMAC-Hash(salt, IKM). An empty salt is RFC 5869-compliant (see hmac_sha256’s key-padding note) — do not special-case it.
