Expand description

VFS integration test cell.

Runs automated test scenarios against the VFS service and prints PASS/FAIL for each. Exit code 0 = all pass, 1 = at least one failure.

Spawn with: spawn /bin/vfs-test from the shell.

All paths use /tmp (RamFS) so the cell runs with or without a block device. The quota tracker in dispatch.rs charges every write regardless of backend, so /tmp quota tests are equivalent to /data quota tests.
