Module gicd 

Source
Expand description

Trap-emulated GICv2 Distributor (GICD) shadow register model.

Security contract (M5, Red Team): this module ONLY maintains a shadow register file. It NEVER issues direct MMIO writes to the physical GICD at 0x08000000. Physical GIC access stays exclusively with the HAL GIC driver in the kernel.

Linux probes GICD during gic_init_bases to read TYPER (IRQ count) and IIDR (implementer). It then programs ISENABLER/IPRIORITYR/ITARGETSR for each IRQ. We emulate just enough for a clean init: writes are accepted silently; reads return safe/plausible values.
