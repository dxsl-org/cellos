Module virtqueue

Split-virtqueue device-side ring processor.

The VMM is on the device side: it reads the avail ring, walks desc chains, hands buffers to the device model, then writes the used ring. All guest memory accesses go via crate::vmm syscall wrappers — never raw pointer dereference from the cell (Law 4).
