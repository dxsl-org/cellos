virtio-blk device model (DeviceID=2, virtio-mmio slot 1 → SPI 17).

16 MiB in-memory backing; reads and writes are volatile (not persisted).

Chain layout (virtio-blk spec §5.2.6.1): [0] outhdr 16B device-readable: { type:u32, _:u32, sector:u64 } [1..n-1] data device-readable (OUT) or device-writable (IN) [last] status 1B device-writable: 0=OK 1=IOERR 2=UNSUPP
