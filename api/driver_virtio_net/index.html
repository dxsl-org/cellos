VirtIO NIC Driver Cell — Tier-1 Privileged Driver Cell.

Probes VirtIO MMIO slots for a Network device (device type 1), claims exclusive MMIO via sys_request_mmio, initialises the virtqueues via virtio-drivers, registers as the system NIC via sys_register_nic_driver, then serves Tx/Rx/GetMac IPC from the net service. This cell replaces the kernel’s virtio_net.rs driver per the Kernel Boundary Law (2026-06-23).

§Fallback

If no VirtIO MMIO NIC is found (x86_64, real hardware with PCIe-only NIC) the cell exits cleanly (code 0). The kernel’s fallback path and the e1000 Driver Cell remain.

§Law 4 exception

src/device.rs uses unsafe for the virtio_drivers::Hal impl and MMIO probe reads. All other modules forbid unsafe code.
