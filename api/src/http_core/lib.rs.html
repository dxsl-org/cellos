1//! Pure, host-testable HTTP/1.1 + JSON protocol logic.
2//!
3//! # Running the host tests
4//!
5//! The workspace `.cargo/config.toml` pins a bare-metal default target
6//! (`riscv64gc-unknown-none-elf`), so a bare `cargo test -p http-core` tries to
7//! build the test harness for `no_std` and fails.  Pass an explicit HOST target:
8//!
9//! ```text
10//! cargo test -p http-core --target x86_64-pc-windows-msvc   # Windows host
11//! cargo test -p http-core --target x86_64-unknown-linux-gnu # Linux host
12//! ```
13//!
14//! (substitute your platform's host triple — `rustc -vV | grep host`).
15//!
16//! # Why this crate exists
17//!
18//! `ostd` owns `#[global_allocator]` and `#[panic_handler]`, and its workspace
19//! `.cargo/config.toml` forces `riscv64gc-unknown-none-elf` → `cargo test -p ostd`
20//! never runs on the host.  All pure byte-in/byte-out protocol code lives here
21//! instead, so that `cargo test -p http-core` runs on the host with the standard
22//! test harness.  `ostd` depends on this crate and adds only transport glue.
23//!
24//! # no_std contract
25//!
26//! Outside of `#[cfg(test)]`, this crate is `no_std` + `alloc`.  It defines
27//! **no** allocator and **no** panic handler — those are the host runner's
28//! (test) or ostd's (bare metal) responsibility.
29#![cfg_attr(not(test), no_std)]
30
