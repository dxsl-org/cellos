Expand description

Core widget trait.

Every UI element implements this. Container widgets (Column, Row in P04) hold Vec<Box<dyn ViWidget>> and recurse into children in layout(), paint(), and event().

Lifecycle (OrbTK State ref): on_mount() → once when inserted into tree. layout() → each frame (or when dirty). post_layout() → after layout, before paint. Widget knows its final bounds. paint() → render. event() → input routing. on_unmount() → once when removed from tree (Law 8: RAII).

fn layout(&self, constraints: Constraints) -> LayoutNode

Compute layout for this widget + all children given constraints.

Returns screen-space LayoutNode. constraints.origin is the top-left of the allocated slot; the returned bounds must fit within constraints.max.

fn post_layout(&mut self, bounds: Rect)

Called after layout(), before paint(). Widget knows its final bounds.

Use cases: ScrollArea clamps scroll_y after knowing visible height; TextEdit computes cursor pixel position after knowing text box width.
