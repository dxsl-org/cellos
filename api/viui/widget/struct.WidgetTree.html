Expand description

Retained widget tree with layout cache, state store, and focus manager.

Elm rebuild strategy: WidgetTree::rebuild(root) is called after each app.update(). The new root replaces the old one cheaply (just a pointer swap). WidgetStateStore (egui Memory) persists across rebuilds via hash-stable WidgetIds.
