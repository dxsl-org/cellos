Overlay layer — shared action queue + entry types for Dialog, DropDown, Toast.

§Design

Widgets cannot mutate ViApp directly (borrow conflict: root.event() holds &mut self.root while tick_with_dt holds &mut self). Instead, widgets close over an OverlayActionQueue clone and push OverlayAction items into it during their event() call. After all events are processed, ViApp drains the queue and executes the deferred actions.
