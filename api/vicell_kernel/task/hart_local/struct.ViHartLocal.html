Expand description

Per-hart local state.

LAYOUT IS ABI — the trap.S reads kernel_tp_for_cells by hardcoded offset and the field order is FIXED. Add new fields AFTER existing ones. #[repr(C)] ensures Rust does not reorder or pad unexpectedly.

Fields§

§hart_id: usize

This hart’s id (0 = boot hart).

§current_cell_id: AtomicUsize

Cell ID currently running on this hart. 0 = kernel (no quota limit).

§kernel_gp: usize

Value of gp captured at install() time — handed to new cells.

§kernel_tp_for_cells: usize

Value of tp that cells inherit on context switch. Currently 0 (cells have no TLS); Phase 05 may give each cell a private tp.

§ready: Spinlock<BTreeMap<u8, VecDeque<usize>>>

Per-hart ready queues keyed by priority (u8; higher = higher priority). Leaf lock: may be locked while holding SCHEDULER, never the reverse.

§current_task_id: AtomicUsize

Task ID currently running on this hart. 0 = idle. Written by this hart only (in pick_next); read by others for zombie-reap.
