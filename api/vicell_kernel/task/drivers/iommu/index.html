Module iommu

Expand description

IOMMU common API — three-phase DMA isolation.

Phase 1 init() — probe hardware, allocate page tables, stay passthrough. Phase 2 map_dma() — drivers register each DMA buffer’s physical range. Phase 3 activate_isolation() — switch from passthrough to enforced page-table mode.

Call order in main.rs: iommu::init() → driver DMA allocs (call map_dma()) → iommu::activate_isolation()
