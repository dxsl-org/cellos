Module pcie_ecam 

Source
Expand description

PCIe ECAM (Enhanced Configuration Access Mechanism) walker.

Scans bus 0, devices 0-31, probing functions per the header-type multi-function bit. Exposes a kernel-internal snapshot of discovered PCI devices.

ECAM bases per machine (QEMU defaults — hardcoded for v1; ACPI MCFG parse and DTB pci-host-ecam-generic reg lookup are documented follow-ups): x86_64 q35 : 0xB000_0000 RISC-V virt : 0x3000_0000 ARM64 virt : 0x3F00_0000

Only bus 0 is mapped (1 MiB). Devices on buses > 0 require extending the MMIO window — documented follow-up (no QEMU NVMe lands on bus > 0).

Call ordering: platform::init() → init_kernel_paging*() → pcie_ecam::init().
