Expand description

One request/reply exchange with service_tid, recv masked to it.

send_buf encodes the request; recv_buf receives the reply and backs the returned slice (caller-owned so the borrow outlives the call). The reply is accepted only if it came from service_tid — a message from any other sender (e.g. a queued input event, Spec 17 §2) is an IpcError::WrongSender, not a decode of the wrong bytes.
