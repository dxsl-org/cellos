Struct GrantHandle 

Source 
pub struct GrantHandle<T> { /* private fields */ }
Expand description

A typed, linear handle to a kernel-managed grant region.

T is a logical element type — the kernel manages raw byte pages and has no knowledge of T. The region holds len / size_of::<T>() elements.

§Linear type invariant

!Copy + !Clone. At most one GrantHandle<T> exists per grant_id in this cell’s address space. This is enforced by Rust’s affine type system (move semantics), not by hardware.

§Drop

Drop calls sys_grant_free, releasing the frames back to the kernel. Use into_raw to transfer ownership without freeing.

pub fn alloc(count: usize) -> Option<Self>

Allocate a new grant region holding count elements of type T.

The kernel allocates page-aligned contiguous frames; the actual region may be slightly larger than count * size_of::<T>() due to alignment.

Returns None on OOM or if count * size_of::<T>() overflows.

Source

pub unsafe fn from_raw(id: usize, len: usize) -> Self

Wrap a raw grant ID received from another Cell via IPC.

§Safety

id must be a valid grant owned by this cell (e.g. via sys_grant_share and the receiver’s confirmation). len must equal the byte length returned by the original sys_grant_alloc. Calling this twice with the same id creates two owners and will double-free on drop — undefined behaviour.

Source

pub fn into_raw(self) -> (usize, usize)

Consume the handle, returning (grant_id, byte_len) without freeing.

Use when passing grant ownership to another Cell via IPC. The receiver must call GrantHandle::from_raw to re-wrap it.

pub unsafe fn as_bytes_mut(&mut self) -> &mut [u8]

Get an exclusive byte slice over the entire grant region.

§Safety

No other live reference to this grant must exist. The handle must still be valid (not freed). The caller must ensure the data at [0, len) is initialised before reading.
