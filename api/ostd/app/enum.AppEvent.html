Expand description

Typed event delivered to an AppContext handler.

AppEvent owns its payload data so the handler can store or forward it without lifetime constraints tied to the receive buffer.

New variants may be added in future SDK releases. Always add a wildcard arm _ => {} to future-proof your match.

§

Init

Fires exactly once before the first sys_recv when using AppContext::run_with_lifecycle or CellRuntime::run. Use for startup work: config read, service registration, UI init.

§

Message

An App SDK message (envelope magic verified).

Fields

§sender_tid: usize

TID of the sending cell.

§data: Vec<u8>

Message payload (bytes after the 2-byte header), heap-owned.

§

RawMessage

A raw IPC message not using the App SDK envelope (legacy / non-SDK senders).

Fields

§sender_tid: usize
§data: Vec<u8>

Full message bytes, heap-owned.

§

Input(InputEvent)

A keyboard or mouse event delivered by the input service.

Automatically decoded from the 65-byte 0x10-opcode IPC message. Cells must call AppContext::request_input_focus at startup to receive these events.

§

Timeout

The receive deadline elapsed (only from AppContext::run_with_timeout).

§

Shutdown

Kernel-requested graceful shutdown (legacy form — no reason byte in envelope).

§

ShutdownWith

Structured shutdown with reason (emitted when the envelope carries a reason byte).

Fields

§reason: ShutdownReason

Why the shutdown is happening.

§

Snapshot

Kernel-sent signal to the OLD cell during Step 2 of a hot-swap sequence.

The cell must serialize its state, stash it under swap_id as the key via [ostd::syscall::sys_state_stash], then yield or exit. The kernel polls the stash; once the bytes appear the swap proceeds to Step 3.

Default arm: cells that do not implement hot-swap state transfer may ignore this event — the swap continues with an empty stash (no state restored).

Fields

§swap_id: u64

Monotonically increasing swap identifier assigned by the kernel orchestrator. Must be used as the stash key so the replacement instance can recover it.

§

Restore

Kernel-sent signal to the NEW cell during Step 4 of a hot-swap sequence.

The cell must restore its state from the stash under key via [ostd::syscall::sys_state_restore], then call [ostd::syscall::sys_hotswap_ready] to signal completion.

key is a null-terminated UTF-8 string encoding the decimal swap_id, held in a fixed 64-byte buffer. Parse with core::str::from_utf8(&key[..key.iter().position(|&b| b==0).unwrap_or(64)]) and convert via str::parse::<u64>().

Fields

§key: [u8; 64]

Fixed-size stash key (null-terminated, decimal swap_id).
