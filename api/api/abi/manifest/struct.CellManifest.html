Expand description

Fixed-layout capability manifest (v2). ABI-stable — see Law 1.

Always 16 bytes due to #[repr(C)] and explicit reserved fields. Version the struct via MANIFEST_VERSION before adding fields.

Fields§

§magic: u32

Must equal MANIFEST_MAGIC; from_bytes rejects any other value.

§version: u8

MANIFEST_VERSION (2) for a native v2 manifest; a v1 upcast keeps this at 2 (the value in tier records that it came from v1).

§tier: u8

Isolation tier request (TIER_*); TIER_LEGACY when upcast from v1.

§flags: u16

Bitwise-OR of MANIFEST_FLAG_* constants (u16 in v2).

§cap_args_off: u32

RESERVED — must be 0. Future offset into a __ViCell_cap_args section for parameterized capabilities; kept here so filling it is an additive section parse, not a third ABI confirmation.

§reserved: u32

RESERVED — must be 0.

pub fn from_bytes(bytes: &[u8]) -> Option<Self>

Parse a manifest from raw ELF section bytes. Field-by-field — never casts the slice to &Self (alignment hazard in no_std). Accepts both the v2 16-byte record and the legacy v1 8-byte record (upcast).

§Returns

None if the bytes are too short, the magic mismatches, the version is unsupported, a reserved field is non-zero, the tier is out of range, or a flag bit outside MANIFEST_FLAGS_MASK is set.
