Hypha core — the agent brain (P3: fs + sys + spawn tools).

Spawns llm-gateway, tool-fs, tool-sys, and tool-spawn, then loops: read a line from stdin (UART), keep the conversation in heap, run an agentic turn (LLM + optional tool sub-loop), print the final reply.

Tool routing by name prefix:

  • fs tools (read_file / write_file / list_dir) → tool-fs
  • sys tools (list_cells / sys_info / lookup_service) → tool-sys
  • spawn tools (spawn_cell / kill_cell) → tool-spawn

render_prompt 🔒
Flatten the heap conversation into a single role-tagged transcript. The system preamble is prepended so the LLM always knows about tools. Trimmed from the front if it would exceed the one-message IPC budget.
run_turn 🔒
Run one agentic turn: call the LLM, dispatch any tool requests, and return the final text reply. Loops up to MAX_TOOL_ROUNDS times to handle chained tool calls. Tool interactions are appended to working_prompt in-place (not stored in the permanent conversation — only the final reply is kept).
