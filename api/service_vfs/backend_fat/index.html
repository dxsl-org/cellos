Module backend_fat 

Source
Expand description

FAT backend over the VirtIO block device (fatfs crate + CachedBlockStream).

Path contract: receives absolute VFS paths and strips self.prefix itself. Two strip modes mirror the pre-MountTable dispatch exactly:

  • ops (write/unlink/…) require a NON-EMPTY relative path — operating on the mount root is rejected;
  • list/stat accept the mount root (/data → volume root).

fatfs uses interior mutability, so all operations go through &self internally; &mut self on the trait is only for RamFS symmetry.
