Pending async read table for the VFS service.

Implements the server side of the two-opcode non-blocking read protocol:

  1. ReadAsync { path } → VFS reads file data synchronously (disk is still blocking), stores it under a handle, returns PendingHandle(id).
  2. Poll { handle: id } → returns Data(bytes) (always ready with synchronous backend) or Err if the handle is stale/consumed.

The protocol is correct regardless of the backend being synchronous: the caller-side loop with yield_now() cooperates correctly with the scheduler, and the API shape is ready for a future interrupt-driven block driver.
