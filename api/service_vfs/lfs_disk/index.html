Module lfs_disk 

Source
Expand description

littlefs storage driver over the VirtIO block device — MBR partition P4.

Byte offsets from littlefs are translated to absolute LBAs inside P4 (api::disk::PART_LFS_BASE_LBA); the kernel’s check_block_access gate independently confines this cell to P1+P4, so even a littlefs bug cannot reach the cell table or snapshot regions.

POWER-SAFETY INVARIANT: every read/write goes straight to the block layer — littlefs’s copy-on-write correctness depends on its prog/erase ordering reaching the device. Do NOT route this through the FAT PageCache. Since the G2 loader redesign, “the block layer” is the block Driver Cell (via blk_router), not the kernel — the raw sys_blk_* syscalls reach only the kernel’s dead null device on QEMU, so every /data write failed until this was routed through blk_router like the FAT backend.
