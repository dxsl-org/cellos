Expand description

Input routing and mouse position state owned by the compositor.

Fields§

§input_tid: usize

TID of the input service cell (0 = not yet connected).

§mouse_x: i32

Logical mouse cursor position (updated from MouseMove events).

§mouse_y: i32§focused_owner: usize

TID of the cell currently receiving keyboard events (0 = no focus, events dropped).

Starts at 0 — events are silently dropped until a surface calls request_input_focus. This matches the input service Dispatcher’s own initial state.

§hw_cursor: bool

True when the VirtIO GPU hardware cursor was successfully uploaded at startup. When true, MouseMove issues GpuCursor(move) instead of triggering a full software repaint via pending_dirty.
